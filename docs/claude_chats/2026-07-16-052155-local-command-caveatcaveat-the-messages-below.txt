  Après l'upload de mon-nouveau-fichier.txt, les compteurs du haut réagissent

  Le vrai problème : le faux backend de démo ne trie pas

  Tu testes contre le mock MSW (VITE_USE_MOCK=true). Or son store diverge du
  vrai backend :

  - register() ajoute le nouveau fichier en fin de tableau : files.push(...)
  (test/mocks/store.ts:45).
  - listFiles() pagine sans trier (store.ts:99-112) : il fait
  filtered.slice(page*size, …) directement, sans aucun sort.

  Résultat : le nouveau fichier finit en queue de tableau → dernière page. La
  liste paraît triée par date uniquement parce que les 8 fichiers de démo sont
  déjà insérés dans l'ordre décroissant — pure coïncidence du seed.

  Le vrai backend, lui, trie bien par date décroissante (FileJpaRepository →
  Sort.by("createdAt").descending() et InMemoryFileMetadataRepository →
  Comparator.comparing(createdAt).reversed()). Donc en production le fichier
  apparaîtrait bien en tête. L'invalidation React (useUploadFiles.onSettled) 
  fonctionne — c'est confirmé par la mise à jour des stats. Le bug est
  uniquement dans le mock de démo.

  Le correctif

  Rendre le mock fidèle au backend en triant par createdAt décroissant dans
  listFiles, avant la pagination :

  let filtered = files.filter((f) => f.status !== 'EXPIRED');
  if (q) filtered = filtered.filter((f) =>
  f.filename.toLowerCase().includes(q));
  if (status) filtered = filtered.filter((f) => f.status === status);
  // Comme le vrai backend : tri par date de creation decroissante, pour que le
  // fichier tout juste uploade (le plus recent) apparaisse en tete de page 0.
  filtered = [...filtered].sort((a, b) => Date.parse(b.createdAt) -
  Date.parse(a.createdAt));

  Une subtilité : le mock fige now() à une date constante, donc plusieurs
  fichiers uploadés partagent le même createdAt. Avec le tri seul (stable), ils
  s'empilent en tête dans l'ordre d'upload — parfait pour un upload simple. Si
  tu veux que le dernier uploadé soit toujours pile en position 0 même après
  plusieurs uploads, j'ajoute un tie-breaker sur l'ordre de création.
