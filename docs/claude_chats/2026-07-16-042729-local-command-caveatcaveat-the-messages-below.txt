  Cause racine : le backend n'a aucune configuration CORS

  Les fetches UI → backend échouent avec TypeError: Failed to fetch parce que le
  backend Spring Boot ne renvoie aucun en-tête CORS. Le navigateur bloque donc
  toutes les requêtes vers l'API.

  Ce n'est ni un backend en panne, ni une mauvaise URL : le backend est bien
  joignable (https://praxedo-api-900523019258.europe-west1.run.app).

  Le mécanisme exact

  1. L'UI (praxedo-ui-…run.app) et l'API (praxedo-api-…run.app) sont sur deux 
  origines distinctes → toute requête est cross-origin.
  2. L'UI ajoute un header custom X-API-Key (src/api/client.ts) → cela force un 
  preflight OPTIONS obligatoire.
  3. Côté backend, SecurityConfig.java n'appelle jamais .cors(...) — aucun
  @CrossOrigin, aucun CorsConfigurationSource, rien. Le preflight OPTIONS 
  /api/** tombe directement sur la chaîne d'authentification par clé API → 403,
  sans en-tête Access-Control-Allow-Origin.
  4. Le navigateur voit un preflight qui échoue → il n'envoie même pas le vrai
  GET → « Failed to fetch » → stats vides (« — ») et liste de fichiers vide,
  exactement ce qu'on voit à l'écran.

  Pourquoi ça marche en local et pas en prod ? En dev, VITE_USE_MOCK=true → MSW
  intercepte tout dans le navigateur (même origine, aucun réseau réel). En prod
  le workflow déploie avec VITE_USE_MOCK="false" → vrais fetches cross-origin →
  bloqués.

  Correctif proposé (ciblé, sans sur-ingénierie)

  1. Backend — SecurityConfig.java : activer CORS dans le SecurityFilterChain
  (.cors(Customizer.withDefaults())) + exposer un bean CorsConfigurationSource
  autorisant l'origine de l'UI, les méthodes utilisées et le header X-API-Key.
  Avec l'intégration native Spring Security, le CorsFilter traite le preflight
  OPTIONS avant le filtre d'auth → plus de 403.
  2. Rendre l'origine configurable via une propriété (ex.
  app.cors.allowed-origins) et injecter UI_ORIGIN dans les --set-env-vars du
  deploy-api (Makefile) — actuellement UI_ORIGIN ne sert qu'au bucket GCS,
  jamais à l'app.

  Bonus (bug latent connexe) : deploy/gcs-cors.json a un slash final sur
  l'origine (…run.app/). L'en-tête Origin du navigateur n'en a jamais → cette
  CORS-là (uploads/downloads par URL signée) ne matchera pas non plus. À
  corriger en même temps.
